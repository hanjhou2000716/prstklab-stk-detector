# PRStK 研究版 Beta 操作手冊

這套系統是公開市場資訊、風險教育與量化研究工具；它不會讀取帳戶、不會連接券商，也不會下單。

## 每天你只要看兩個地方

1. Telegram：手錶上的快報會保持在 30 字內，先知道市場是否有顯著變化。
2. [市場儀表板](https://hanjhou2000716.github.io/prstklab-stk-detector/)：點 Telegram 的按鈕或直接開啟網址，閱讀完整資料、研究候選與資料健康度。

快報是提醒；詳細資訊一律以儀表板為準。任何「研究候選」都不是買賣指令。

## 日常排程

系統會在台灣交易日的盤前、盤中、盤後時段執行快報。美股盤前會依紐約時間自動對應：夏令時間為台灣 21:00、冬令時間為台灣 22:00，不會同一天發兩次。

GitHub 排程與 cron-job.org 備援使用同一日期與時段的防重複機制。若某次資料來源不可用，訊息或儀表板會直接顯示「資料暫時無法取得」，不會拿舊資料猜測。

## 想手動跑一次完整台美研究時

1. 進入儲存庫的 **Actions**。
2. 選 **Unified Taiwan-US research report**。
3. 按 **Run workflow**。
4. 第一次建議保留預設值（台股 100、美股 100、每批 50）。
5. 等工作流程顯示綠色勾勾後，回到儀表板查看「台美全市場研究摘要」。

這會做四份公開資料研究：台股／美股動能、台股／美股裸 K 結構。結果還會保存在該次 Actions 的 Artifact，方便下載 CSV 與 JSON。

首次 100+100 的實測約需 7–8 分鐘；資料源當下較慢時可能更久。掃描只是手動研究工具，不會送 Telegram 或影響排程快報。

## 如何閱讀「資料健康度」

| 顯示 | 代表什麼 | 你該做什麼 |
|---|---|---|
| 健康 | 來源可用且報表在 180 分鐘內產生 | 正常閱讀研究摘要。 |
| 需留意 | 有來源缺漏、資料較舊或時間格式異常 | 先看卡片中的具體原因；稍後手動重跑一次。 |
| 本次無研究候選 | 掃描成功但沒有標的符合該策略 | 不需要補資料，也不代表市場一定轉空。 |
| 資料暫時無法取得 | 公開資料來源無回應或批次失敗 | 系統已自動重試一次；可稍後再跑，勿把舊結果當最新資料。 |

## 想做單一標的的歷史研究時

在 Actions 選 **Price Action walk-forward backtest**，輸入公開 Yahoo Finance 代碼：

- 台股例：`2330.TW`，市場選 `taiwan`
- 美股例：`NVDA`，市場選 `us`

預設模式會研究結構風險邊界與 10R；勾選 **Free Roll** 時，研究假設在 5R 記錄半數結果、剩餘半數以成本價作為風險邊界。報表會計入設定的手續費與滑價，並對同日高低點先後無法判斷的情況標示「順序不明」。歷史結果不代表未來表現。

## 安全規則

- Telegram Token、Chat ID、GitHub PAT 只能放在 GitHub Secrets，不能貼進程式碼、Issue、PR 或聊天記錄。
- 不要在系統中填入券商帳密、錢包資訊或銀行資料；這套系統不需要它們。
- 若 Telegram Token 曾外流，先到 BotFather 重新產生，再更新 GitHub Secret。
- 研究配置與再平衡功能只產生人工檢視草案；實際決定與風險承擔仍由你自行負責。
